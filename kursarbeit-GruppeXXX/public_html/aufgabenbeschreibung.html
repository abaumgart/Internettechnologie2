Prototyp-Entwicklung einer JavaScript basierten Internetanwendung,
die Studierende beim Thema "Mental Health und Achtsamkeit" während des Studiums unterstützt.

Die Aufgabenstellung umfasst drei zu realisierende Bereiche, die gemeinsam eine Anwendung ergeben. Jeder Bereich muss eigenverantwortlich von einem Gruppenteilnehmer bearbeitet werden und wird im Anschluss unabhängig voneinander bewertet.
Für alle Anwendungen gelten die gleichen Anforderungen, die allgemein weiter unten beschrieben sind.

Entwickel mit deiner Gruppe Ideen für zwei oder drei unterschiedliche Anwendungen, die Studierende beim Thema "Mental Health und Achtsamkeit" unterstützt. (Du darfst dabei auf deine Ergebnisse aus dem LEGO® SERIOUS PLAY®-Workshop zurückgreifen).
Beschreibe die Funktion / den Nutzen schriftlich auf max. einer Seite und skizziere grafisch die Webseite / Webseiten / Formulare sowie den Ablauf, damit die Funktionalität ersichtlich wird.
Mit „Ablauf“ ist gemeint, dass aus der Beschreibung und Skizze ersichtlich wird, was wann und in welcher Reihenfolge geschieht.

Realisierung mit JavaScript, HTML und CSS

Detaillierte Anforderungen:

  • Anwendung:
    • Die technische Entwicklung der Anwendung hat eine höhere Priorität als das Design. Kurz: Funktion geht vor Optik.  Trotzdem ist der Entwurf optisch ansprechend zu gestalten. Die Bedienung der Anwendung erfolgt im Browser auf dem Desktoprechner oder Notebook. Es darf das Bootstrap-Framework (Bootstrap https://getbootstrap.com) mit erweiterten HTML, CSS oder JavaScript-Funktionalitäten verwendet werden.
    • Es dürfen auch weitere Frameworks bspw. für die Visualisierung von Daten verwendet werden
      • Bspw. GoogleChart (https://developers.google.com/chart/interactive/docs/quick_start?hl=de)
      • Diagrammtypen mit dem PlotlyframeWork: (https://plotly.com/javascript/)
    • Startseite:
      • Enthält mind. drei HTML-Tabellen mit Kerninformationen jeweils aus den zwei / drei Anwendungsbereichen. Es gelten für alle drei Tabellen die folgenden Spezifikationen:
        • Es werden mind. 3 Spalten angezeigt.
      • Enthält ein Navigationsmenü, um alle für die Darstellung notwendigen Seiten aufzurufen.
      • Für die Startseite gilt, dass diese möglichst effizient aufgebaut ist. Nutze dafür modulare JavaScript-Funktionen.
  • Vorgaben:
    • Funktionsumfang:
      • Es muss mind. ein Formular für die Erfassung / Speicherung von Daten realisiert werden.
      • Die Speicherung der Daten geschieht im objektorientierten Array. Im Beispielprojekt wäre das bspw. das Array zentraleDatenStudent1 in der Datei datenStudent1.js
      • Es muss der in der Anforderung beschriebene Funktionsumfang abgebildet werden.
      • Die erfassten Daten müssen mind. in einer Form visualisiert dargestellt werden. Die Klasse googleChart.js darf genutzt werden. Du kannst auch GoogleChart direkt in deine Seite einbinden.
    • Allgemeine Projektstruktur
      • Ihr dürft das Beispielprojekt an eure Anforderungen anpassen. Dabei müssen alle Verzeichnisse, und Beispielcodes, die im Beispiel student1, student2 … heißen umbenannt werden. Das gilt ebenfalls für den Beispielcode, in dem student1 usw. verwendet wird.
      • Strukturierter Projektaufbau bzgl. der Verzeichnisse / Dateinamen.
        • Für eine separate Bewertung ist unterhalb des public_html Ordners bspw. folgende Unterordnerstruktur zu verwenden:
          • public_html
            • 345345abaumgart
            • 985623khoffmann
            • 234543ctholen
        • In diesen Unterordnern befinden sich alle Dateien, die jedes einzelne Teammitglied erstellt und bearbeitet hat. Verlinkungen sind immer relativ anzugeben.
    • Abgabeform:
      • Als komprimierte Datei im Zip-Format
      • PDF-Dokument
        • Deckblatt
        • Stärken und Schwächen der eigenen Anwendung.
    • Grafiken:
      • Grafiken sind ausschließlich im SVG-Format zu verwenden. Die Formatierung erfolgt ausschließlich über CSS.
      • Via JavaScript ist mind. eine Fläche farblich zu ändern. Versuchen Sie, dies über eine function zu realisieren. Welche Parameter müssten Sie dieser mitgeben, damit die function von allen genutzt werden könnte?
      • Werden aufgrund der App-Idee Fotos eingebunden, dürfen diese im jpg- oder png-Format eingebunden werden.
    • Quellcodeverwaltung:
      • Die Nutzung von GitHub ist optional
    • HTML / CSS
      • Alle Designvorgaben müssen mittels CSS spezifiziert werden.
      • Alle Formatierungen am Layout bzgl. Farben, Tabellen etc. sind ausschließlich über Stylesheets zu realisieren.
      • Werden Formulare verwendet, gilt: Alle Listen / Dialogfelder eines Formulars müssen mit dynamischen Inhalten aus einem Array gefüllt werden.
      • Wird die bereitgestellte Klasse "Formularmanager" genutzt, dann wird das Formular in einem objektorientiertem Array definiert. Siehe Beispiel.
    • JavaScript
      • Es muss mind. ein objektorientiertes Array (pro Teilnehmer) genutzt werden, in dem die Kerndaten der Anwendung gespeichert sind. Diese umfasst mind. 5 Attribute. Wird die Projektvorlage genutzt, sind dort schon Arrays vorhanden, die ebenfalls genutzt werden dürfen. Das Array für die "Kerndaten" kommt von der Anzahl zu verwendender Arrays noch hinzu.
      • Die Anzahl der anderen Arrays - wenn benötigt – ergibt sich aus der Anwendung.
      • Es ist ein möglichst modularer Quellcode zu verwenden. Nutze effizient Funktionen (Mit Parametern und Rückgabewerten).
      • Verwende, soweit wie möglich, EventListener.
      • Verwende so wenig HTML-Code wie möglich und generiere ihn alternativ über JavaScript.
